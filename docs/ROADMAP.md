# Roadmap

Owlyn MCP should stay small, local-first, and focused on work-session supervision for AI agents.

## v0.1.0-alpha.2

Focus: public presentation, CI, npm publishing readiness, and docs.

- GitHub Actions CI
- README badges
- npm publishing documentation
- roadmap
- FAQ
- launch post draft
- public docs polish

## v0.1.0-alpha.3

Focus: Codex-first hardening and validation.

- Codex MCP setup guide
- Codex agent instruction guide
- Codex validation scenario matrix
- Codex validation log
- clearer host validation matrix
- keep Cursor and Claude Desktop as future compatibility targets without overclaiming support

## v0.1.0-beta.1

Focus: stability.

- stronger invalid-input tests
- migration robustness
- structured output consistency
- verified config examples
- better error messages
- no major known host issues

## v0.1.0-beta.2

Focus: usability.

- session report polish
- install flow polish
- CLI/helper scripts only if needed
- documentation polish
- npm package readiness

## v1.0.0-rc.1

Focus: release candidate.

- no new features
- bug fixes only
- host validation matrix updated
- npm package tested
- README final pass
- security and limitations reviewed

## v1.0.0

Stable release criteria:

- Codex MCP host validated
- at least one more real MCP host validated
- all 8 tools stable
- tests pass
- CI passes
- npm package published or publish-ready
- documentation complete
- no known critical issues
- limitations clearly stated
- explicit release-owner approval

## Not Before v1.0

Avoid these before v1.0:

- UI
- web dashboard
- cloud sync
- auth
- telemetry
- analytics
- browser extension
- arbitrary file access
- shell execution from MCP tools
